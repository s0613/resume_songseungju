Core Roles

7개 전문가가
안에 있습니다.

각 에이전트는 역할 명세와 독립 프로토콜을 가집니다. Tech Lead가 요청을 분석하고 필요한 전문가만 정확히 투입합니다.

PM
요구사항 분석, 리스크 검토, 복잡도 판단, 우선순위 정의
Design
레퍼런스 DNA 기반 UI 설계, 거부 방향 봉인, 브랜드 정체성 누적
Tech Lead
전문 서브에이전트 병렬 디스패치 + 결과 통합 + 팀 채널 조율
Frontend
UI · 컴포넌트 · 접근성 · 반응형 구현
Backend
API · 서버 · 도메인 로직 설계 및 구현
Security
OWASP Top 10 + STRIDE 구현, cross-cutting 리뷰
QA
구현자 산출물 참조 없이 PM 브리프 기반 독립 검증

What's Different

팀처럼 협업합니다.

에이전트들은 서로 직접 소통합니다. 사람처럼 협력하고, 결과만 돌려줍니다.

01
팀 채널 직접 조율
서브에이전트들은 Tech Lead를 거치지 않고 직접 소통합니다. Database가 "nullable 컬럼 주의"를 게시하면 Backend가 직접 읽고 처리합니다.
02
취향이 쌓이는 디자인
거부한 방향은 봉인되고, 승인한 방향은 누적됩니다. 시간이 지날수록 브랜드 정체성이 선명해집니다.
03
QA 독립성 보장
QA는 구현자 요약을 읽지 않습니다. PM 브리프와 실제 파일을 직접 탐색해 편향 없이 검증합니다.

Commands

/sj-company 하나로
시작합니다.

태스크를 말로 설명하면 라우팅됩니다. 단독으로 호출할 수 있는 전문 커맨드도 있습니다.

/sj-company 모든 것의 시작점 — 태스크를 설명하면 적절한 전문가로 자동 라우팅
/spec 모호한 의도 → 5단계 실행 가능한 정밀 명세
/design 레퍼런스 브랜드 DNA 기반 UI 설계
/investigate 가설 수립 → 검증 강제, 추측성 수정 금지
/cso OWASP + STRIDE 보안 감사 자동화
/ship 테스트 → 커버리지 → PR 자동화
/outsource 막혔을 때 전문가 위임 — PII 마스킹 맥락 리포트 + 메일 초안 자동 작성
